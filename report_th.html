การตรวจจับติ่งเนื้อในลำไส้ — เปรียบเทียบ YOLO รุ่นต่างๆ ข้ามประเภทกล้อง

YOLO รุ่นไหนตรวจจับติ่งเนื้อได้ดีที่สุดเมื่อเปลี่ยนประเภทกล้อง?

PolypDB · เปรียบ YOLO 4 รุ่น · เทรนบนกล้องแสงปกติ (WLI) → ทดสอบกับกล้องพิเศษ 4 ชนิดที่ไม่เคยเห็นมาก่อน

ก่อนอื่น — เราเปรียบอะไรกับอะไร?

ใน Computer Vision มี Model หลายประเภทมากครับ เช่น CNN, U-Net, Transformer แต่ละตัวเหมาะกับงานต่างกัน

โปรเจกต์นี้ใช้ YOLO เท่านั้น — ซึ่งเป็น Model ที่เชี่ยวชาญการตรวจจับและระบุตำแหน่งของวัตถุในภาพแบบ Real-time สแกนรูปแค่รอบเดียวแล้วบอกได้เลยว่า "ติ่งเนื้ออยู่ตรงนี้"

สิ่งที่เราเปรียบคือ YOLO 4 รุ่น ที่มีโครงสร้างภายในต่างกัน: - YOLO26s — รุ่น Custom ที่ดัดแปลงมา - YOLOv11n, YOLOv11s — รุ่นใหม่ล่าสุดจาก Ultralytics (n = เล็ก, s = กลาง) - YOLOv8n — รุ่นก่อนหน้า ยังนิยมใช้กันอยู่

เงื่อนไขการทดลอง: ทุกรุ่นใช้ข้อมูลเดียวกัน เทรนนานเท่ากัน ปรับค่าเหมือนกันทุกอย่าง — เปลี่ยนแค่ตัว Model

คำถามที่ต้องการคำตอบ

กล้องส่องลำไส้ในโรงพยาบาลมีหลายประเภท บางชนิดใช้แสงปกติ บางชนิดใช้แสงพิเศษเพื่อเห็นเส้นเลือดหรือเนื้อเยื่อชัดขึ้น

ภาพจากกล้องแต่ละชนิดมีหน้าตาต่างกันมาก แต่ Dataset ส่วนใหญ่เป็นภาพจากกล้องแสงปกติ (WLI) ถึง 91%

คำถามคือ:

ถ้าเทรน AI ด้วยกล้องแสงปกติอย่างเดียว — YOLO รุ่นไหนสามารถ "นำความรู้ไปใช้กับกล้องชนิดอื่นที่ไม่เคยเห็น" ได้ดีที่สุด?

คำว่า "นำความรู้ไปใช้กับข้อมูลใหม่ได้" ในภาษา AI เรียกว่า Generalize — ถ้า Model generalize ได้ดี แปลว่ามันเข้าใจ "รูปทรงของติ่งเนื้อจริงๆ" ไม่ใช่แค่จำสีและแสงของกล้องตัวที่เทรน

Dataset นี้มีความลำเอียง (Bias) สูงมาก — WLI มีถึง 91% ส่วนกล้องอื่นมีรวมกันแค่ 9% สิ่งที่เกิดขึ้นระหว่างเทรนคือ Model จะเห็น WLI แทบทุก Batch ทำให้มันค่อยๆ "ถูก shape" ไปตาม WLI โดยไม่ได้ตั้งใจ — นี่คือโจทย์หลักที่ต้องรับมือ

ความแตกต่างของแสงแต่ละ Modality

กล้องแต่ละชนิดใช้แสงต่างกัน ทำให้ภาพที่ได้มีสีและหน้าตาต่างกันมาก แม้จะถ่ายติ่งเนื้อเดียวกัน

Modality ประเภทแสง ลักษณะภาพ ความต่างจาก WLI
WLI แสงขาวปกติ สีเนื้อเยื่อตามความเป็นจริง — (ต้นแบบ)
LCI แสงขาวเพิ่มความคมชัด ใกล้เคียง WLI แต่คมกว่าเล็กน้อย ต่ำ
FICE ประมวลผล Narrow-band จำลองความแตกต่างของเนื้อเยื่อ กลาง
NBI 415nm + 540nm สีน้ำเงิน-เขียว เน้นเส้นเลือด สูง
BLI 410nm Narrow-band สีน้ำเงินเข้ม เห็นรายละเอียดสูง สูงสุด

ยิ่งแสงต่างจาก WLI มาก ภาพยิ่งดูแตกต่าง และยิ่งยากสำหรับ Model ที่เทรนจาก WLI — ลำดับนี้จะสะท้อนออกมาในผลการทดลอง

ผลลัพธ์

ก่อนดูผล — mAP@50 และ Avg OOD คืออะไร?

mAP@50 (Mean Average Precision at IoU 50%) คือตัวชี้วัดหลักของการตรวจจับวัตถุ วัดสองสิ่งพร้อมกัน: - "ตรวจเจอไหม?" — Model วาดกรอบมาหรือเปล่า - "กรอบถูกไหม?" — กรอบที่วาดทับกับกรอบจริงมากแค่ไหน

@50 หมายความว่า นับว่า "ถูก" ก็ต่อเมื่อกรอบที่ Model วาดทับกับกรอบจริงอย่างน้อย 50% ถ้าทับน้อยกว่านั้น = ผิด ค่า 1.0 = แม่นสมบูรณ์, ค่า 0 = พลาดหมด

Avg OOD (Average Out-of-Distribution) = ค่าเฉลี่ย mAP@50 ของ 4 กล้องที่ Model ไม่เคยเห็นระหว่างเทรน (LCI + FICE + NBI + BLI) ตัวเลขนี้สำคัญที่สุดในโปรเจกต์นี้ — เพราะมันบอกว่า Model "เอาความรู้ไปใช้กับของใหม่" ได้ดีแค่ไหน ไม่ใช่แค่ท่องจำสิ่งที่เคยเห็น

ภาพรวม Cross-Modal mAP@50

ยิ่งสีเข้ม = ยิ่งแม่น · แถว = Model · คอลัมน์ = ประเภทกล้อง · สังเกตว่า YOLO26s (แถวบนสุด) จางลงชัดเจนตั้งแต่ FICE เป็นต้นไป

สิ่งที่ค้นพบ

1. Model ใหญ่กว่าไม่ได้แปลว่าดีกว่า

YOLO26s กับ YOLOv11s มีขนาดใกล้กันมาก (9.47M vs 9.41M Parameters) แต่ผลต่างกันมาก

Model WLI → BLI ตก WLI → NBI ตก
YOLO26s −22.2% −19.3%
YOLOv11n −5.6% −4.8%
YOLOv11s −4.3% −3.9%
YOLOv8n −5.0% −5.7%

YOLO26s ตกหนักกว่า YOLOv11s ถึง 4-5 เท่า ทั้งที่มีขนาดเท่ากัน

สาเหตุที่เป็นไปได้: YOLO26s เรียนรู้จาก WLI จนจำ "สีและ Texture" ของกล้องแสงขาวได้ดีมาก แต่พอกล้องเปลี่ยนเป็น NBI (สีน้ำเงิน-เขียว) หรือ BLI (สีน้ำเงินเข้ม) มันคลาดเคลื่อนและตรวจพลาด ต่างจาก YOLOv11s ที่ดูเหมือนเข้าใจ "รูปทรงของติ่งเนื้อ" มากกว่าสี จึงยังทำงานได้แม้แสงจะเปลี่ยนไป

2. YOLOv11s ดีที่สุดโดยรวม

  • ค่าเฉลี่ยกล้องที่ไม่เคยเห็น (Avg OOD): 95.9%
  • LCI ได้สูงสุด 99.2% — เพราะ LCI ใช้แสงที่ใกล้เคียง WLI มากที่สุด Model จึงแทบไม่สับสน
  • สม่ำเสมอทุก Modality ไม่มีจุดอ่อนชัดเจน

3. YOLOv8n คุ้มค่าที่สุด

YOLOv8n ใช้ Parameters แค่ 3.01M — น้อยกว่า YOLOv11s ถึง 3 เท่า แต่ได้ผล 95.3% Avg OOD ห่างกันแค่ 0.6% ถ้าต้องการ Deploy บนอุปกรณ์ที่มีพลังประมวลผลจำกัด YOLOv8n น่าสนใจมาก

4. ลำดับการตกเหมือนกันทุก Model

ทุก Model ตกตามลำดับ: LCI น้อยสุด → FICE → NBI → BLI มากสุด

ลำดับนี้ตรงกับความต่างของแสงพอดี — LCI แสงใกล้ WLI มากที่สุด BLI แสงต่างมากที่สุด แสดงว่าการตกที่เห็นมาจาก "ความต่างของกล้อง" จริงๆ ไม่ใช่ข้อบกพร่องของ Model ที่แก้ได้ด้วยการเลือก Architecture ต่างกัน

Hyperparameter Tuning — ทดลองปรับค่าสี (HSV Ablation)

ใน YOLO มี parameter ชื่อ HSV ที่ควบคุมว่าระหว่างเทรน Model จะเห็น "สีที่สุ่มเปลี่ยนไปมา" มากแค่ไหน เพื่อไม่ให้มันติดกับสีแบบใดแบบหนึ่งจนเกินไป

ในการเทรนหลัก ผมเปิด HSV=0.7 ไว้ (ค่า default) เพื่อให้ Model เห็นสีหลากหลายระหว่างเทรน — hypothesis คือถ้ามันเห็นสีหลายแบบ มันจะไม่ยึดติดกับสีของ WLI และ generalize ข้าม modality ได้ดีขึ้น

แต่ในงานวิจัยด้าน Medical Imaging มีแนวคิดตรงข้ามว่า "สีของเนื้อเยื่อคือข้อมูลที่สำคัญ" เช่น สีแดง = อักเสบ สีซีด = สภาพเนื้อเยื่อ ถ้าสุ่มเปลี่ยนสีตลอด Model อาจเรียนรู้ที่จะ ไม่ใช้สีในการตัดสินใจ ซึ่งอาจทำให้พลาด feature สำคัญที่แพทย์ใช้จริง

เลยทดลองเทรน YOLOv11s อีกรอบโดย Lock สีไว้ที่ค่าจริง (HSV=0) แล้วเปรียบผล

ผลการทดลอง

Modality HSV=0.7 (เปิดสุ่มสี) HSV=0 (Lock สี) ดีกว่า
WLI 0.983 0.949 HSV=0.7
LCI 0.992 0.931 HSV=0.7
FICE 0.961 0.933 HSV=0.7
NBI 0.943 0.872 HSV=0.7
BLI 0.940 0.975 HSV=0 ← ข้อยกเว้น
Avg OOD 0.959 0.928 HSV=0.7

ผลโดยรวม HSV=0.7 ชนะใน 4 จาก 5 Modality แต่ BLI คือข้อยกเว้นที่น่าสนใจ

คำอธิบายเบื้องต้น: BLI ใช้แสงสีน้ำเงิน Narrow-band ที่ต่างจาก WLI มากที่สุด เมื่อ Lock สีไว้ Model อาจหันไปพึ่งพารูปทรง (Shape) มากขึ้น ซึ่งอาจช่วยได้ในกรณีที่สีต่างกันสุดขั้ว

สิ่งที่ต้องระวังก่อนสรุปว่า "HSV=0.7 ชนะ"

ผลข้างบนดูชัด แต่มีประเด็นที่ต้องคิดให้รอบก่อนสรุปเด็ดขาด

1. ผลนี้มาจาก seed เดียว — อาจเป็นความบังเอิญ

YOLO training มีความแปรปรวนตามธรรมชาติ เทรนสองรอบด้วยข้อมูลเดิมแต่ seed ต่าง ตัวเลขก็อาจต่างกัน ±2–4% ได้โดยไม่มีอะไรเปลี่ยน

ส่วนต่างระหว่าง HSV=0.7 กับ HSV=0 บน BLI คือ 3.5% ซึ่งอยู่ใน range ของ variance ปกติ ดังนั้น "BLI exception" อาจไม่ใช่ signal จริง แต่เป็นแค่ผลของ seed นั้น — ต้องรัน 3–5 seeds แล้วดู average ถึงจะสรุปได้มั่นใจ

2. คำอธิบายเรื่อง Shape ยังไม่ได้พิสูจน์จริง

ที่บอกว่า "HSV=0 พึ่ง Shape มากขึ้น" ยังเป็นแค่ hypothesis ไม่ใช่สิ่งที่ตรวจวัดได้จากตัวเลขนี้ตรงๆ

วิธีพิสูจน์ที่ถูกต้องคือทำ EigenCAM / Grad-CAM ซึ่งเป็น visualization ที่แสดงว่า Model "มองไปที่ส่วนไหนของภาพ" ก่อนตัดสินใจ ถ้า HSV=0 พึ่ง Shape จริง จะเห็น attention กระจุกที่ขอบ (edge) ของติ่งเนื้อ ถ้ายังพึ่งสี จะเห็นกระจุกที่ texture ของเนื้อเยื่อ — นี่คือ next step ที่ควรทำก่อนสรุป

3. NBI ขัดแย้งกับคำอธิบายที่ให้ไป

ถ้า "กล้องที่ต่างจาก WLI มาก → HSV=0 ทำได้ดีกว่า" เป็นกฎจริง NBI ก็ควรแสดง pattern เดียวกับ BLI เพราะ NBI ก็ใช้แสง Narrow-band และต่างจาก WLI มาก

แต่จริงๆ บน NBI — HSV=0.7 ยังชนะตามปกติ (0.943 vs 0.872)

นั่นแปลว่าคำอธิบายที่ให้มายังไม่ครอบคลุมทุกกรณี มีปัจจัยอื่นที่ยังไม่ได้อธิบาย

4. เราปิด HSV ทั้งหมดพร้อมกัน ทั้งที่ควรทดสอบทีละแกน

HSV ใน YOLO จริงๆ แยกเป็น 3 แกน: - H (Hue) = เฉดสี เช่น แดง น้ำเงิน เขียว - S (Saturation) = ความเข้มของสี - V (Value) = ความสว่าง

การที่ NBI/BLI ใช้แสง Narrow-band หมายความว่ามันต่างจาก WLI ที่ Hue เป็นหลัก ไม่ใช่ Saturation หรือ Brightness

ดังนั้นถ้าจะทดลองอย่างตรงประเด็น ควรปิดทีละแกน แทนที่จะปิดหมดพร้อมกัน — อาจเป็น Hue augmentation ที่ขับ generalization จริงๆ ไม่ใช่ Saturation/Value

5. Average OOD อาจไม่ใช่ metric ที่ hospital ต้องการจริงๆ

เราสรุปว่า HSV=0.7 ชนะเพราะ Average OOD ดีกว่า แต่ในการ Deploy จริง โรงพยาบาลแต่ละแห่งใช้กล้องยี่ห้อเดียว เช่น Olympus (NBI) หรือ Fujifilm (BLI) ไม่ได้ใช้ครบ 5 ชนิดในห้องเดียว

ถ้าโรงพยาบาลเป้าหมายใช้ BLI เป็นหลัก → HSV=0 คือตัวที่ดีกว่าสำหรับเขา ทั้งที่ average แพ้

ยิ่งถ้าคิดในแง่ Medical Screening — การพลาดติ่งเนื้อ 1 เม็ดมีโอกาสกลายเป็นมะเร็งในอนาคต ดังนั้น Recall สูง (ตรวจเจอให้ครบ) อาจสำคัญกว่า Average mAP ในบางบริบท

ข้อสรุปที่ซื่อสัตย์ที่สุด: จากการทดลองนี้ HSV=0.7 น่าจะเป็นตัวเลือกที่ดีกว่าโดยรวม แต่ยังมีคำถามที่เปิดค้างไว้หลายข้อ — โดยเฉพาะ BLI exception ที่ต้องการ seed เพิ่ม และ EigenCAM เพื่อยืนยัน hypothesis ก่อนสรุปได้อย่างมั่นใจ

วิเคราะห์ข้อผิดพลาด

ทำไม YOLO26s ถึงทำได้ไม่ดีที่สุด ทั้งที่เป็นรุ่นใหม่สุด?

YOLO26s ทำ WLI ได้ 98.3% เท่ากับทุกตัว แต่พอกล้องเปลี่ยน ผลตกหนักมาก นั่นแปลว่ามันไม่ได้ "ไม่เก่ง" — มันเก่งมากแต่เก่งแบบผิดวิธี มันจำ "หน้าตา" ของ WLI ไว้แม่น แต่ไม่ได้เข้าใจว่าติ่งเนื้อมีรูปทรงยังไง

เปรียบได้กับคนที่จำได้ว่าหมาตัวที่บ้านมีขนสีน้ำตาล พอเจอหมาขาวก็จำไม่ออก ต่างจากคนที่เข้าใจว่า "หมา" มี 4 ขา มีหาง มีหู ซึ่งจำได้ไม่ว่าสีอะไร

ช่องว่างที่เหลือใน YOLOv11s

ที่ YOLOv11s ยังตกอยู่บ้างบน NBI/BLI ไม่ใช่ความผิดพลาดของ Model ครับ มันคือ "ขีดจำกัดจริง" ของการเทรนด้วยกล้องชนิดเดียว ต้องแก้ด้วยการเพิ่มข้อมูลจาก NBI/BLI เข้าไปในการเทรน ไม่ใช่การเปลี่ยน Architecture

ข้อจำกัดและสิ่งที่ต้องทำต่อ

จากการทดลอง HSV Ablation พบว่า HSV=0.7 ให้ผลดีกว่า HSV=0 ในเกือบทุก Modality ยกเว้น BLI ที่ HSV=0 ทำได้ดีกว่า ซึ่งน่าสนใจมากเพราะ BLI คือ Modality ที่แตกต่างจาก WLI มากที่สุด

ข้อจำกัดของโปรเจกต์นี้:

  1. เทรนแค่ WLI เท่านั้น — ถ้าเพิ่มข้อมูลจาก NBI หรือ BLI เข้าไปในการเทรน น่าจะช่วยปิดช่องว่างที่เหลืออยู่ได้มาก
  2. Test set ของกล้องชนิดอื่นมีน้อยมาก — BLI และ LCI มีแค่ 7-9 รูปใน Test set ตัวเลขที่ได้อาจไม่เสถียรพอที่จะสรุปได้อย่างมั่นใจ 100%
  3. Tune ได้แค่ HSV — ยังไม่ได้ทดลองปรับ Learning Rate, จำนวน Epoch ที่มากขึ้น หรือค่าอื่นๆ
  4. ยังไม่ได้วิเคราะห์ทีละภาพ — รู้แค่ตัวเลขรวม ยังไม่รู้ว่า Model พลาดรูปแบบไหนบ้าง เช่น ติ่งเนื้อขนาดเล็ก หรือบริเวณขอบภาพ
